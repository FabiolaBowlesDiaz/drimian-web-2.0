**1. Create `src/islands/` directory and `Diagnostic.tsx`:**

The component uses `useReducer` for state management with the DiagnosticState/Action types from types.ts.

**Reducer logic** (port from research Pattern 2):
- OPEN: phase='question', currentQuestion=0, answers=new Array(10).fill(null), showRevelation=false
- ANSWER: set answers[questionIndex]=value, showRevelation=true
- DISMISS_REVELATION: if currentQuestion < 9, advance to next question (currentQuestion+1, showRevelation=false, phase='question'); if currentQuestion === 9, phase='results'
- GO_BACK: if currentQuestion > 0, go back (currentQuestion-1, showRevelation=false, phase='question')
- CLOSE: phase='idle', clear sessionStorage
- RESTORE: replace entire state with provided state

**Overlay structure:**

Add two features to the existing Diagnostic.tsx:

**1. History API integration (DIAG-06):**

Per research Pattern 4 and Pitfall 2:

- When diagnostic opens (OPEN action): `history.pushState({ diagnosticQuestion: 0 }, '', window.location.pathname)`
- When advancing to next question (after DISMISS_REVELATION): `history.pushState({ diagnosticQuestion: nextQ }, '', window.location.pathname)`. Do NOT change the URL.
- When going back via UI button: do NOT pushState. Just dispatch GO_BACK.
- When going back via browser back button: listen for `popstate` event.

```typescript
useEffect(() => {
  const handlePopState = (e: PopStateEvent) => {
    if (state.phase === 'idle') return; // ignore if diagnostic not open
    if (e.state?.diagnosticQuestion !== undefined) {
      dispatch({ type: 'GO_BACK' });
    } else {
      // User backed out entirely
      dispatch({ type: 'CLOSE' });
    }
  };
  window.addEventListener('popstate', handlePopState);
  return () => window.removeEventListener('popstate', handlePopState);
}, [state.phase]);
```

CRITICAL per Pitfall 2: Only pushState when ADVANCING forward. Never push when going back. Use replaceState if user re-answers a previous question (though current design doesn't allow re-answering).

When overlay closes (CLOSE action), if there are history entries for the diagnostic, we need to clean up. Simplest: just let popstate handle it. If user presses X, dispatch CLOSE. If they then press back, popstate fires but phase is idle so we ignore it.

**2. sessionStorage persistence (DIAG-07):**

Per research Pattern 6 and CONTEXT.md decision:

- On every state change (answers or currentQuestion change), save to sessionStorage:
```typescript
useEffect(() => {
  if (state.phase !== 'idle') {
    sessionStorage.setItem('diagnostic-state', JSON.stringify({
      phase: state.phase,
      currentQuestion: state.currentQuestion,
      answers: state.answers,
      showRevelation: state.showRevelation
    }));
  } else {
    sessionStorage.removeItem('diagnostic-state');
  }
}, [state]);
```

- On component mount, check for saved state and restore if mid-diagnostic:
```typescript
useEffect(() => {
  const saved = sessionStorage.getItem('diagnostic-state');
  if (saved) {
    try {
      const parsed = JSON.parse(saved);
      if (parsed.phase && parsed.phase !== 'idle') {
        dispatch({ type: 'RESTORE', state: parsed });
        // Also lock body scroll since we're restoring an open diagnostic
        document.body.style.overflow = 'hidden';
        document.body.dataset.diagnosticOpen = 'true';
      }
    } catch { /* corrupted state, ignore */ }
  }
}, []);
```

- CONTEXT.md decision "No resume, starts fresh": When user clicks "Empezar el diagnostico" (OPEN action), it ALWAYS starts fresh from question 1 (the reducer already does this). The sessionStorage restore only happens on page refresh/accidental navigation -- not on intentional re-entry.

- When CLOSE is dispatched, clear sessionStorage: `sessionStorage.removeItem('diagnostic-state')`.
